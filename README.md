# portal-wcschedule-guide

## 项目简介

`portal-wcschedule-guide` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定的域名或网站，仅作为一组静态页面的集中存放与版本管理。

这些页面可能包含指南、说明、参考表格或工具页面，主要用于信息查阅与快速访问。

## 目录结构

```
portal-wcschedule-guide/
├── docs/               # 存放各 HTML 页面及其相关资源
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 共用的样式、脚本或图片文件（可选）
│   ├── styles.css
│   └── script.js
├── README.md           # 本文件
└── index.html          # 可选的仓库首页或导航页
```

## 页面归档说明

- 所有页面均为独立的 HTML 文件，不依赖后端服务。
- 各页面之间无强制关联，可单独打开使用。
- 页面内容可能随时间更新或替换，历史版本可通过 Git 提交记录追溯。
- 若页面需要外部资源（如 CSS、JS、图片），建议统一放在 `assets/` 目录下，以保持结构清晰。

## 使用方式

你可以通过以下方式访问本仓库中的页面：

1. **直接浏览**：克隆仓库到本地后，在浏览器中打开 `docs/` 目录下的任意 HTML 文件。
2. **GitHub Pages 部署**：若仓库启用了 GitHub Pages，可通过对应 URL 访问 `docs/` 目录下的文件（具体路径取决于部署设置）。

## 维护说明

- 新增页面请放置在 `docs/` 目录下，并保持命名规范。
- 若页面引用外部资源，请确保资源路径正确（建议使用相对路径）。
- 不建议在页面中嵌入敏感信息或动态内容，本仓库仅适合存放纯静态页面。
- 提交更新时，请附上清晰的提交信息，便于追踪变更。
